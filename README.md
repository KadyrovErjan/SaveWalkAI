## 📌 Что это такое

**SaveWalk AI** — это система помощи при ходьбе, которая:

- 📷 Захватывает видео с камеры в реальном времени
- 🧠 Обнаруживает людей и объекты через нейросеть **YOLOv8**
- 📏 Рассчитывает расстояние до каждого объекта
- 🔊 Озвучивает предупреждение голосом когда объект **приближается**
- 🎯 Отслеживает каждый объект по уникальному **track_id** (ByteTrack)

Основная цель — помогать людям с ограниченными возможностями зрения безопасно передвигаться в городской среде.

## ⚙️ Принцип работы

```
Камера (OpenCV)
     │
     ▼
YOLO + ByteTrack          ← обнаружение + присвоение track_id
     │
     ▼
raw_distance()            ← pin-hole расчёт сырого расстояния
     │
     ▼
DistanceSmoother          ← moving average (N=5) + bucket + hysteresis
     │                       0.7м→bucket=0 (молчим)
     │                       1.0м→bucket=1 (ГОВОРИМ "1 метр")
     │                       1.2м→bucket=1 (молчим — не изменилось)
     ▼
SoundManager              ← антиспам + cooldown + threading
     │
     ▼
🔊 sounds/person/2.wav    ← воспроизведение без блокировки FPS
```

### Защита от спама — 7 уровней

| # | Условие | Описание |
|---|---------|----------|
| 1 | Класс объекта | Только `person` (настраивается) |
| 2 | Зона оповещения | Только если `distance < 5м` |
| 3 | Bucket > 0 | Не озвучиваем "менее 1 метра" |
| 4 | Стабильность | Объект виден ≥ 3 кадра подряд |
| 5 | Изменение bucket | `2м → 1м` да, `1.1м → 1.3м` нет |
| 6 | Cooldown | Минимум 4 секунды между озвучками |
| 7 | Наложение | Новый звук не запускается пока играет текущий |

## 🚀 Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/KadyrovErjan/SaveWalkAI.git
cd SaveWalkAI
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r req.txt
```

### 4. Запустить

```bash
python main.py
```

Нажми **`Q`** для выхода.

## 🔊 Как добавить свои звуки

Файлы называются по номеру метра: `1.wav`, `2.wav`, `3.wav` ...

```
sounds/
  person/
    1.wav   ← озвучивается когда человек на расстоянии 1.0 – 1.99 м
    2.wav   ← 2.0 – 2.99 м
    3.wav   ← 3.0 – 3.99 м
```

Если точного файла нет — система возьмёт ближайший меньший.

## 🎛 Настройка

Все параметры находятся в `sound_manager.py` в классе `SoundManager`:

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `TRIGGER_CLASSES` | `{"person"}` | Классы объектов для озвучки |
| `MAX_DIST_M` | `5.0` | Максимальная дистанция оповещения (м) |
| `STABLE_FRAMES` | `3` | Кадров подряд для "стабильного" объекта |
| `COOLDOWN_SEC` | `4.0` | Пауза между озвучками одного объекта |
| `RETRIGGER_AFTER` | `6.0` | Сброс cooldown если объект пропал (сек) |

Калибровка камеры — в `distance.py`:

```python
# Поставь человека на 2.0 м, замерь ширину bounding box в пикселях
# FOCAL_LENGTH = (pixel_w * 2.0) / 0.5
FOCAL_LENGTH = 700   # подбери под свою камеру
```

## 🗺 Архитектура озвучки (bucket-логика)

Вместо `round()` используется `int()` + hysteresis:

```
round(0.7) = 1  ← НЕПРАВИЛЬНО (говорит "1 метр" при 0.7м)
int(0.7)   = 0  ← ПРАВИЛЬНО   (молчит до 1.0м)

Bucket таблица:
  0.00 – 0.99 м  →  bucket 0  (молчим)
  1.00 – 1.99 м  →  bucket 1  → 1.wav
  2.00 – 2.99 м  →  bucket 2  → 2.wav
  ...

Hysteresis = 0.15 м:
  Вошли в bucket=1 при 1.0 м
  Выходим обратно только при < 0.85 м
  Дребезг 0.95 ↔ 1.05 → не срабатывает
```
